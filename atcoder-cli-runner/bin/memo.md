# PowerShell スクリプト文法 チートシート

## 1. 変数の基礎

PowerShellでは、`$` から始まるものはすべて変数として扱われます。

### ユーザー定義変数

自分で名前を決めて値を保存する変数です。

- `$repoRoot` や `$Session` など、名前は自由に付けられます。

### 自動変数（システム変数）

PowerShellがあらかじめ用意している特別な変数です。

- **`$ErrorActionPreference`**: エラー発生時の基本動作を決める変数。`"Stop"` を代入すると、エラー時に即座にスクリプトを停止する（デフォルトは
  `"Continue"`）。
- **`$PSScriptRoot`**: **現在実行しているスクリプトファイルが存在するフォルダのパス**。
- **`$HOME`**: 現在のユーザーのホームディレクトリ（例: `C:\Users\Username`）。
- **`$null`**: 値が存在しないこと（null）。
- **`$true` / `$false`**: 真偽値（boolean）。

### 特殊変数

- **`$_`**（または `$PSItem`）: パイプラインを流れてきた、現在処理中のデータ。
- **`$env:`**: Windowsの環境変数に直接アクセスするためのプレフィックス。
	- 例: `$env:Path`（現在のPATH環境変数）、`$env:USERNAME`（ユーザー名）

## 2. 記号と基本構文

### .NET Framework の機能呼び出し `[Class]::Method()`

PowerShellの裏側で動いているWindowsのコア機能（.NET）を直接呼び出す強力な記法です。

- **`[ ]` (ブラケット)**: クラス。
- **`::` (コロン2つ)**: そのクラスが持つ「静的メソッド」や「プロパティ」。
	- 例: `[Environment]::GetEnvironmentVariable("Path", "User")`
	- 例: `[System.Net.NetworkCredential]::new("", $secure).Password`（暗号化文字列の平文戻し）

### 文字列と変数展開

- **`"..."`**: 中に変数を書くと、実行時にその値に展開される。
	- 例: `"$HOME\.atcoder\session.txt"` → `C:\Users\xxx\.atcoder\session.txt`
- **`'...'`**: 中に何が書いてあっても、**ただの文字列**として扱われる。

### 配列（リスト）

- **`@()`**: 配列を宣言・初期化します。
	- 例: `$list = @()` （空の配列）
	- 例: `$list = @(1, 2, 3)`

### パイプライン `|`

- 左側の処理結果を、右側のコマンドに「入力データ」としてそのまま渡す。

## 3. 演算子

PowerShellの演算子は、記号ではなくハイフン (`-`) から始まるアルファベットで記述します。

### 比較演算子

| 演算子       | 意味 (略称の由来)            | 使用例        |
|-----------|-----------------------|------------|
| **`-eq`** | 等しい (Equal)           | `$a -eq 1` |
| **`-ne`** | 等しくない (Not Equal)     | `$a -ne 1` |
| **`-gt`** | より大きい (Greater Than)  | `$a -gt 1` |
| **`-ge`** | 以上 (Greater or Equal) | `$a -ge 1` |
| **`-lt`** | より小さい (Less Than)     | `$a -lt 1` |
| **`-le`** | 以下 (Less or Equal)    | `$a -le 1` |

### 文字列・正規表現マッチング

| 演算子                        | 意味                                                |
|----------------------------|---------------------------------------------------|
| **`-like` / `-notlike**`   | ワイルドカード (`*`, `?`) と一致する / しない                    |
| **`-match` / `-notmatch**` | 正規表現と一致する / しない                                   |
| **`-replace`**             | 正規表現で見つけた文字列を置換する（例: `$str -replace "^REV=", ""`） |

### コレクション・論理演算子

| 演算子                | 意味                | 使用例                           |
|--------------------|-------------------|-------------------------------|
| **`-contains`**    | 配列の中に特定の値が含まれているか | `$list -contains "apple"`     |
| **`-in`**          | 特定の値が配列の中に含まれているか | `"apple" -in $list`           |
| **`-join`**        | 配列の要素を指定した文字で結合する | `$list -join ";"`             |
| **`-not` / `!`**   | 否定。               | `if (!$isOk) { ... }`         |
| **`-and` / `-or`** | かつ / または          | `if ($a -eq 1 -and $b -eq 2)` |

## 4. よく使う標準コマンドレット (Cmdlet)

PowerShellの標準命令は `動詞-名詞` の分かりやすい形式になっている。

### 出力・入力系

- **`Write-Host`**: コンソールに文字を表示する。
- 便利オプション: `-ForegroundColor Green`


- **`Write-Output`**: 文字を画面に出しつつ、後続のパイプラインにデータを流す。
- **`Read-Host`**: ユーザーにキーボード入力を促す。
- 便利オプション: `-AsSecureString` (入力文字を `***` で隠す)

### ファイル・フォルダ操作系

- **`Test-Path`**: 指定したファイルやフォルダが存在するかをチェックし `$true` / `$false` を返す。
- **`New-Item`**: 新しいファイルやフォルダを作成。
	- 例: `New-Item -ItemType Directory -Path $dir -Force`


- **`Set-Content`**: ファイルにテキストを上書きする。
- **`Get-Content`**: ファイルの中身を読み込む。

### コレクション操作系 (パイプラインと組み合わせる)

- **`Where-Object`** (エイリアス: `?`): 条件に一致する要素だけをフィルタリングする。
	- 例: `$list | Where-Object { $_ -ne "" }`


- **`ForEach-Object`** (エイリアス: `%`): 渡ってきたデータすべてに同じ処理を実行する。
	- 例: `$list | ForEach-Object { Write-Host $_ }`
